# ISSM Scheduled Maintenance Proposal

`ISSM_Scheduled_Maintenance_Proposal_Template.docx` — the formal proposal submitted by the SA to the ISSM for written authorization of scheduled maintenance.

## When to Use
Submit this document to your ISSM **before** any scheduled maintenance cycle. The ISSM must sign off prior to execution of any tasks listed.

## Sections
1. Purpose and Scope
2. Personnel and Roles
3. Maintenance Windows
4. Scheduled Maintenance Tasks (by periodicity: Daily → Annual)
5. Risk and Impact Assessment
6. Configuration Change Control
7. Continuous Monitoring Artifacts
8. Exceptions and Deviations
9. SA Notes and ISSM Comments
10. Approval and Sign-Off

## Workflow
1. SA completes all bracketed fields.
2. SA attaches relevant MRCs for each task listed.
3. SA submits to ISSM for review.
4. ISSM marks decision (Approved / Approved with Conditions / Denied) and signs.
5. Signed proposal is retained as a ConMon artifact.
