# Category 05 — Account & Access Management


## JSIG Applicability
This category maps to JSIG-required controls. All tasks require written ISSM authorization (MA-2) and are retained as Body of Evidence (BoE) artifacts for the ATO package and ConMon program.

## Applicable JSIG / NIST SP 800-53 Controls
| Control | Title | JSIG Requirement |
|---------|-------|-----------------|
| AC-2 | Account Management | Required — provisioning, monitoring, termination |
| AC-3 | Access Enforcement | Required — enforce approved authorizations |
| AC-5 | Separation of Duties | Required — prevent conflict of interest |
| AC-6 | Least Privilege | Required — minimum necessary access |
| IA-4 | Identifier Management | Required — manage user IDs across lifecycle |
| IA-5 | Authenticator Management | Required — manage passwords, CAC/PKI |

## JSIG-Specific Notes
- All privileged accounts must be tied to an individual — no shared accounts (JSIG / AC-2).
- Dormant privileged accounts must be disabled after [X] days per policy.
- ISSM must maintain a list of authorized maintenance personnel per MA-5.
- AC-6(1) is NON-TAILORABLE — privileged access to security functions must be tightly controlled.
- Account reviews are BoE artifacts for AC-2 and CA-7 ConMon.
