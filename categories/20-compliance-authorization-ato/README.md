# Category 20 — Compliance & Authorization (ATO)


## JSIG Applicability
This category maps to JSIG-required controls. All tasks require written ISSM authorization (MA-2) and are retained as Body of Evidence (BoE) artifacts for the ATO package and ConMon program.

## Applicable JSIG / NIST SP 800-53 Controls
| Control | Title | JSIG Requirement |
|---------|-------|-----------------|
| CA-5 | Plan of Action and Milestones | Required — track and remediate weaknesses |
| CA-6 | Security and Privacy Authorization | Required — ATO granted by AO; track expiration |
| CA-7 | Continuous Monitoring | Required — ongoing security monitoring |
| RA-3 | Risk Assessment | Required — assess risks and document |
| PL-2 | System Security Plan | Required — current SSP required for ATO |

## JSIG-Specific Notes
- ATO is granted by the SAP Authorizing Official (AO) — not ISSM. ISSM coordinates and recommends.
- ATO renewal must be initiated at least 90 days before expiration.
- Continuous monitoring data (all MRCs and ConMon artifacts) feeds the ATO renewal package.
- ISSM maintains all cybersecurity documentation including ATOs (DoDM 5205.07).
- POA&M must be reviewed and updated monthly — open POA&M items may affect ATO status.
