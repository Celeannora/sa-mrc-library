# Category 14 — Incident Response Readiness


## JSIG Applicability
This category maps to JSIG-required controls. All tasks require written ISSM authorization (MA-2) and are retained as Body of Evidence (BoE) artifacts for the ATO package and ConMon program.

## Applicable JSIG / NIST SP 800-53 Controls
| Control | Title | JSIG Requirement |
|---------|-------|-----------------|
| IR-3 | Incident Response Testing | Required — periodic testing/exercises |
| IR-4 | Incident Handling | Required — capability to handle incidents |
| IR-6 | Incident Reporting | Required — report to ISSM, PSO, SAPCO as applicable |
| IR-8 | Incident Response Plan | Required — develop, maintain, test IR plan |
| IR-9 | Information Spillage Response | Required — SAP environments require specific spillage response procedures |

## JSIG-Specific Notes
- JSIG requires IR-9 (Information Spillage Response) — SAP spillage reporting is a distinct, serious obligation.
- SAP spillage events: GSSO/CSSO reports to PSO via secure communications; PSO reports to Director of Security and CA SAPCO.
- IR plan must be reviewed after any actual incident, not just on schedule.
- ISSM coordinates all inspections, tests, and reviews (DoDM 5205.07 ISSM responsibilities).
